Spotkanie projektowe 07.05.2015
===============================
- mapa kafelkowa, plain text, schemat pliku poniżej (specyfikacja pliku w innej sekcji)
- poruszanie gracza pomiędzy kaflami jest "płynne", tzn może być jednocześnie na 2 polach (gracz jest mniejszy niż kafel)
- logika i grafika zostanie oddzielona od siebie: klasy zawierające logikę nie posiadają elementów, które można wyświetlać
- zaczynamy grę od pisania prostego rdzenia: cała gra nie jest jeszcze podzielona na 2 osobne aplikacje, tzn część klientową i serwerową
- code style guide zostanie podany do waszej weryfikacji jeszcze przed pisaniem pierwszego kodu



MAP FILE FORMAT
---------------
Obecna wersja formatu: 1.0

Pliki map są zapisywane jako Plain Text (w trybie tekstowym) i składają się z sekcji:
[header]
[version]
[size_w] [size_h]
[data]

Każda sekcja rozpoczyna się w osobnej linii i zawierająco następująco
- [header] (string); stała część każdego pliku, zawierająca magiczną wartość "BOMBERMAN_LEVEL", 
   w razie braku tego nagłówka należy przerwać wczytywanie pliku i zwrócić błąd
- [version] (float); informacja o wersji formatu (przydatne w momencie wczytania poziomu zawierającego nowsze niewspierane wcześniej feature'y), 
   jeżeli wersja nie jest wspierana przez parser należy zwrócić błąd, w przeciwnym wypadku skonwertować plik do nowszej wersji
- [size w] [size_h]; szerokość i wysokość poziomu jako liczba całkowita z zakresu [0,0xffff]
- [data] (uint16_t); sekcja danych zawierające o kaflach w postaci liczb całkowitych z zakresu [0, 0xffff]; kolejne numery są oddzielone białymi znakami (spacja lub nowa linia)


Przykłady:
(example1.txt):
BOMBERMAN_LEVEL
1.0
3 3
0 0 0
1 2 3
0 0 0


(exapmple2.txt):
BOMBERMAN_LEVEL
1.0
5 2
0 0 0 1 1
0 2 1 1 1

FAZY PROJEKTU
-------------
Każdy Milestone składa się z kilku faz:
1. Faza Projektowania - wstępne wymyślenie featerów, rozplanowanie pracy teamu
2. Faza Kodzenia - pisanie pojedynczych modułów
3. Faza Testowania - poddanie modułu z (2) testom i ocenie reszcie teamu, w razie problemów powrót do (2)
4. Faza Łączenia - finalizacja modułu poprzez połączenie go z resztą projektu (merge do master) i przejście do kolejnego modułu
